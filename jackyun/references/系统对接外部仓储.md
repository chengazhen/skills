对接前准备

请吉客云使用者（商家）提供下述4个参数，找吉客云开放平台技术发起京东虎符上线申请，等第三方仓储接受该申请后，方可进行后续配置操作。

参数1: 提供第三方仓储的appkey（找第三方仓储要）

参数2: 商家名称

参数3 ：商家PIN

参数4 ：提供线上环境回调地址，在下图界面中获取(新建委外托管仓储-线上回调地址)。

当上述操作完成后，需要第三方仓储提供以下几个数据给商家使用。

数据1：客户ID 和货主编码（一般值为一样）,用于接口类型

数据2: 仓库编码（可能多个）,用于接口类型

数据3：物流方式编码（可能多个）,用于物流匹配

public HttpParms GetHttpParms(string method,string customerid, string dliveryRequest){    

       var dicRequest = new SortedDictionary<string, string>() {

              {"method",method},

       };

       //生成签名md5后转大写

ContentType = "application/x-www-form-urlencoded;charset=utf-8"

//md5算法

        public static string MakeMD5(string original, Encoding encoding){

           byte[] bytes =   encoding.GetBytes(original);          

            var hashmd5 = new MD5CryptoServiceProvider();

          byte[] byteOriginal = hashmd5.ComputeHash(bytes);

            StringBuilder ciphertext = new StringBuilder(32);

            for (int i = 0; i < byteOriginal.Length; i++) {

                ciphertext.Append(byteOriginal[i].ToString("x").PadLeft(2, '0'));

            }

return ciphertext.ToString();

        }

    public String encrypt(String text, String encoding) throws Exception    {

        if (encoding == null || encoding.isEmpty()) {

            encoding = "UTF-8";

        }

        MessageDigest md5 = MessageDigest.getInstance("MD5");

        byte[]

        resultByte = text.getBytes(encoding);

3.6 入库单确认接口:

接口名称：entryorder.confirm

请求参数:

<?xml version="1.0" encoding="utf-8"?>

<request> 

  <entryOrder> 

<totalOrderLines>单据总行数，int，当单据需要分多个请求发送时，发送方需要将totalOrderLines填入，接收方收到后，根据实际接收到的条数和totalOrderLines进行比对，如果小于，则继续等待接收请求。如果等于，则表示该单据的所有请求发送完成。

</totalOrderLines>

    <entryOrderCode>入库单编码,  string (50) ,  必填</entryOrderCode>  

    <ownerCode>货主编码,  string (50) </ownerCode>  

    <warehouseCode>仓库编码, string (50)，必填 </warehouseCode>   

    <entryOrderId>仓储系统入库单ID, string (50) , 条件必填</entryOrderId>  

    <entryOrderType>入库单类型 ，SCRK=生产入库，LYRK=领用入库，CCRK=残次品入库，CGRK=采购入库, DBRK=调拨入库, QTRK=其他入库，B2BRK=B2B入库</entryOrderType>  

    <outBizCode>外部业务编码, 消息ID, 用于去重, ISV对于同一请求，分配一个唯一性的编码。用来保证因为网络等原因导致重复传输，请求不会被重复处理, ,必填</outBizCode>  

<confirmType>支持出入库单多次收货, int，

多次收货后确认时

0 表示入库单最终状态确认；

1 表示入库单中间状态确认；

【网店订单下载】>【网店订单转审核】>【销售单审核】>【递交仓库发货】>【发货在途】

1. 订单下载

（1）在【目录】>【销售渠道】>【渠道列表】页面新建一个直营网店。

                 

（2）在【销售】>【网店订单】>【网店订单下载】页面对店铺进行授权。因为各个平台授权流程有差异，可以点击【查看授权指引】参考对应的授权流程。

                 

（3）授权完成后，将店铺后台的订单下载到吉客云中。下载订单有两种方式：

手动下载

        

自动下载：在策略配置中开启自动下载

        

2. 订单处理流程 

（1）订单下载下来后，将网店订单转入审核，在【销售单审核】界面将发货仓库改为菜鸟中心仓。（该步骤可建仓库匹配策略进行自动分配仓库）

（2）审核完订单后，在仓库发货界面点击“递交仓库发货”，将订单推送给菜鸟中心仓，若订单状态变为“待发货-已递交”，则表示订单递交成功。

                 

（3）菜鸟中心仓发货完成后，状态订单会自动变为“发货在途”，物流信息会更新到订单中。

                

四、常见对接问题

        问题：操作库存同步的时候，提示请求失败。

        

        答案：商家首次使用需要菜鸟进行参数配置，提供货主id给菜鸟技术进行配置后方可使用。

问题：提交仓库失败，提示【预占库存失败，无当前条件可操作库存】。

答案：请先在菜鸟GOS系统中进行库存分组划拨，将库存划拨至【非淘toC】渠道。如果库存已经划拨，说明此商品在仓库的库存不足，请在仓库进行补货后再进行递交。

问题：订单在菜鸟中心仓已经发货成功了，但是在吉客云中订单没有显示物流信息。

答案：在【物流公司】界面添加一家物流公司（菜鸟仓发货采用的物流公司），然后到【存货仓库】>【数据同步】>【物流匹配】找到已经添加的物流公司填入承运商编码。

        

问题：订单提交仓库后提示buyIDNO这个字段为空或者非法。

答案：检查一下销售单详情中证件类型、姓名、证件号是否有填写。

问题：订单提交仓库后提示payOrderId这个字段为空或者非法。

答案：检查一下销售单中支付单号是否有值。如果没有，请将销售单驳回到网店订单，重新下载订单后再转入销售单审核进行发货。

问题：订单提交仓库后提示传入参数不合法Paytype。

答案：此销售单的支付方式菜鸟中心仓不支持，请先在菜鸟中心仓将此支付方式备案后进行递交。

问题：订单递交仓库发货失败，提示【仓储平台返回】系统错误，没有显示具体错误原因

答案：打开订单详情排查一下是否有未合并的商品条目。如有重复货品可以尝试将订单驳回到【销售单审核】，选中该订单点击【编辑】，并在修改订单界面点击【合并重复货品】后保存即可。

问题：首次递交仓库发货，提示平台请求失败。

答案：检查一下订单对应的销售平台菜鸟是否支持。电商平台需要在海关做过备案，并且在菜鸟关务做过备案，才能在菜鸟中心仓发货。

问题：请求平台失败(请求平台失败，附加消息：【仓储平台返回】:授权有误，请检查绑定的应用是否正确。) 

答案：检查一下在物流云控制台授权的应用的appkey是否是506557，如果授权错误，请重新授权再进行递交。

问题：订单提交仓库后提示"商品交易时hscode与备案hscode不一致且增值税税率不一致"。

答案：请在【存货仓库-接口类型-数据同步-货品同步】中重新同步货品再进行递交。
